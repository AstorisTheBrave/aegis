# Maintainer program

Maintainers may submit connectors, policy packs, workflow templates, SDK fixes,
fixtures, and documentation. Every ecosystem artifact needs a named maintainer,
immutable source revision, reproducible build digest, passing test evidence,
declared compatibility, and lifecycle state.

Connector maintainers must retain least-privilege scope evidence and prove that
their contribution cannot issue provider writes under protocol v1. Reviewers may
request updated fixtures, compatibility evidence, or a deprecation plan before
accepting a release.
